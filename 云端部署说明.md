# 分享墙 · 线上远程协作部署说明

要实现「像网盘一样」在线上打开链接、多人随时发表和修改，有三种常见做法：

| 方式 | 难度 | 费用 | 适合 |
|------|------|------|------|
| **A. 云端版 + Supabase** | ★★☆ | 免费额度 | 推荐，无需自己买服务器 |
| **B. 部署服务器版** | ★★★ | 约 ¥30+/月 | 数据完全在自己机器上 |
| **C. 内网穿透** | ★★☆ | 免费~低价 | 临时用，电脑要一直开着 |

---

## 方式 A：云端版（推荐）

### 思路

- 页面放在网上（或本地打开 `分享墙-云端.html`）
- 文字、链接、图片存在 **Supabase**（免费云数据库 + 图床）
- 所有人访问**同一个网址 + 同一个房间号**，即可实时看到彼此的内容

### 第一步：注册 Supabase

1. 打开 https://supabase.com 注册（可用 GitHub 登录）
2. 新建项目 **New project**，记下区域和密码

### 第二步：初始化数据库

1. 左侧 **SQL Editor** → **New query**
2. 打开本项目 `cloud/supabase-setup.sql`，复制全部内容粘贴执行
3. 左侧 **Storage** 确认有名为 `share-board` 的桶，且为 **Public**

### 第三步：获取连接信息

1. **Project Settings** → **API**
2. 复制：
   - **Project URL**（形如 `https://xxxx.supabase.co`）
   - **anon public** 密钥（`eyJ...` 开头）

### 第四步：使用云端分享墙

1. 用浏览器打开 `分享墙-云端.html`
2. 首次打开在顶部填入 URL、anon 密钥，点「保存并连接」
3. 设置**房间号**（如 `family2025`），把**同一房间号**告诉同伴
4. 可选：把页面部署到 **GitHub Pages / Vercel / 腾讯云静态托管**，得到固定 https 链接

### 第五步：分享网址给他人

部署示例（GitHub Pages）：

1. 把 `share-board` 文件夹推到 GitHub 仓库
2. 仓库 **Settings** → **Pages** → Source 选 `main` 分支
3. 访问 `https://你的用户名.github.io/仓库名/分享墙-云端.html`
4. 告诉对方：网址 + 房间号 +（若你写了）Supabase 配置可预填在页面里

> **安全提示**：anon 密钥会出现在前端，当前策略为「知道链接的人可读写」。仅在小范围分享；勿公开到搜索引擎。日后可在 Supabase 加登录（Auth）收紧权限。

## 方式 B：买云服务器部署 Python 版

适合希望数据完全在自己服务器、不依赖 Supabase。

1. 购买 **阿里云 / 腾讯云轻量服务器**（Linux）
2. 安装 Python 3，上传 `share-board` 文件夹
3. 执行：

```bash
pip install -r requirements.txt
# 用 nohup 或 systemd 常驻运行
python app.py
```

4. 安全组放行 **3000** 端口
5. 访问 `http://服务器公网IP:3000`
6. 建议用 **Nginx** 反代 + 免费 **HTTPS**（Let's Encrypt）

数据在服务器 `data/board.db` 和 `uploads/` 目录。

## 方式 C：本机运行 + 内网穿透（临时）

1. 本机双击 `start.bat` 或 `python app.py`
2. 使用 **花生壳 / ngrok / frp** 把 3000 端口映射到公网
3. 把生成的 `https://xxx.ngrok.io` 发给他人

缺点：你的电脑要一直开机。

## 常见问题

**Q：和网盘有什么区别？**  
A：网盘是存文件；分享墙是「可编辑的图文链接墙」。云端版把内容存在数据库里，通过网页增删改查。

**Q：多人会互相覆盖吗？**  
A：每人编辑**自己的那条**不会冲突；同时改同一条以后保存的为准。可点「刷新」拉取最新列表。

**Q：国内访问 Supabase 慢怎么办？**  
A：可改用方式 B 国内服务器；或使用国内 BaaS（如腾讯云开发 CloudBase），思路相同。
