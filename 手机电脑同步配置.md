# 手机电脑同步 · 5 分钟配置

你之前用的是 **本机版**（`分享墙.html`），数据存在各自浏览器里，所以手机发的内容电脑看不到。

要 **远程发表并共享**，请用 **云端版** + **Supabase**（免费）。

---

## 第一步：注册 Supabase

1. 打开 https://supabase.com 注册
2. **New project** 新建项目，记住密码

## 第二步：建表

1. 左侧 **SQL Editor** → **New query**
2. 复制 GitHub 仓库里 `cloud/supabase-setup.sql` 的全部内容，粘贴运行
3. 左侧 **Storage** 确认有 **share-board** 桶（Public）

## 第三步：复制密钥

**Project Settings → API**，复制：

- **Project URL**
- **anon public** 密钥

## 第四步：写入网站配置

在 GitHub 仓库编辑 **`cloud-config.js`**：

```javascript
window.SHARE_BOARD_CLOUD = {
  url: "https://你的项目.supabase.co",
  key: "eyJ...你的anon密钥...",
  board: "main",
};
```

保存后 GitHub Desktop **Commit → Push**，等 1～2 分钟 Pages 更新。

## 第五步：手机、电脑打开同一链接

```
https://nanbian525.github.io/share-board-Nanbian525/
```

（首页已自动跳转到云端版）

首次若未自动连接，在页面填 URL、密钥、房间号 `main`，点「保存并连接」。

---

## 验证是否成功

1. 手机发布一条文字
2. 电脑打开同一网址，点「刷新」或等 30 秒
3. 能看到 → 配置成功

---

## 分享给他人

连接成功后点 **「复制分享链接」**，发给对方即可（同一房间号）。

---

## 常见问题

**连接失败 / relation does not exist**  
→ 没运行 `supabase-setup.sql`，回第二步。

**图片上传失败**  
→ Storage 里确认 `share-board` 桶为 Public，且 SQL 里 storage 策略已执行。

**仍看到本机版**  
→ 确认访问的是首页或 `分享墙-云端.html`，不是 `分享墙.html`。
